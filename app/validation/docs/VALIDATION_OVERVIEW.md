# FFR Insights — Output Validation Overview

Every piece of text generated by the AI goes through a series of automated safety and accuracy checks **before** it reaches the user. This document explains what those checks do, why they matter, and what happens when something fails.

**Applies to both outputs:**
- **Summary** — the financial health report (metric summaries, pillar summaries, overall summary)
- **Insight cards** — short actionable observations shown to the user

---

## What happens when a check fails?

| Output type | Failure behaviour |
|-------------|-------------------|
| **Summary** | The AI is asked to regenerate with specific correction instructions. This retry loop runs up to 3 times. If it still fails, the request returns an error instead of showing bad content. |
| **Insight card** | The individual card is silently dropped. Other clean cards still show. |

In both cases, every failure and retry is logged for audit.

---

## 1. Regulatory & Compliance Guardrails

**Why it matters:** Indian financial-services regulations prohibit giving direct financial advice, recommending specific products, or making guaranteed-return claims in an analytical report. These checks ensure we never cross that line.

**Applies to:** Summary and Insight cards

### Blocked language (high risk — always rejected)

| Category | What it catches | Examples |
|----------|----------------|----------|
| Prescriptive advice | Telling the user what to do | "you should invest", "you must reduce spending", "we recommend" |
| Product solicitation | Pushing specific financial products | "apply for a loan", "open an account", "invest in mutual funds" |
| Product ranking | Declaring one product the best | "best investment option", "top fund", "ideal plan" |
| Guaranteed returns | Promising certain outcomes | "guaranteed returns", "risk-free", "100% safe", "will definitely grow" |
| Return promises | Promising profits | "double your money", "earn high returns", "wealth quickly" |
| Risk minimization | Downplaying investment risk | "no loss possible", "principal protected", "never lose money" |
| Tax evasion language | Aggressive or illegal tax claims | "avoid tax", "tax loophole", "tax hack" |
| Legal advice | Practising law | "consult a lawyer", "you must file", "legally required" |
| Judgmental profiling | Labelling the user | "you are poor", "financially irresponsible", "spending addict" |
| Fear and urgency | Alarmist or pressure language | "urgent action needed", "critical problem", "act now" |

### Flagged language (medium risk — logged for review, not blocked)

| Category | What it catches | Examples |
|----------|----------------|----------|
| Soft advice | Gentle nudges that could be seen as advice | "consider reducing", "you may want to", "could benefit from" |
| Behavioural judgment | Characterising user behaviour negatively | "overspending", "wasting money", "lifestyle inflation" |
| Implied recommendation | Indirect advice disguised as analysis | "this means you should", "the best move is" |

### Insight-only: fallback text leak

If the AI fails to generate a real insight and falls back to placeholder text like "Insufficient data available" or "No insight generated", the card is dropped.

## 2. Number Accuracy (Grounding)

**Why it matters:** The AI sometimes invents or miscalculates numbers. Every number shown to the user must trace back to their actual financial data. A wrong number erodes trust instantly.

**Applies to:** Summary and Insight cards

### How it works

Every number in the output (rupee amounts, percentages, credit scores, ratios) is compared against the user's actual input data. The system checks:

- **Exact match** — the number appears directly in the data
- **Near match** — the number is within a small tolerance (e.g. ±2% for percentages)
- **Derivable** — the number can be calculated from the data (e.g. sum of two monthly values, an average, a ratio)

### What is checked — Summary

| Metric | What is verified |
|--------|-----------------|
| Spend-to-income ratio | Percentages match the ratio series; rupee amounts match income or spend series |
| Credit score | Any 3-digit score cited must appear in the credit score history |
| EMI burden | Percentages match the EMI burden series; large amounts match monthly EMI values |
| Investment rate | Percentages match the investment rate series; amounts match monthly investment values |
| Life insurance cover | Percentages match the adequacy ratio; amounts match current or ideal cover |
| Health insurance cover | Same as life insurance, for health cover data |
| Emergency corpus | Validates the corpus-to-ideal ratio, percentage vs benchmark, month count, and rupee amounts |
| Portfolio diversification | Allocation percentages must match a slice from the portfolio breakdown |
| Portfolio overlap | If no overlap data exists, the output must say so (not invent a percentage) |
| Tax savings | "N out of 5" must match the tax saving index score |
| Savings consistency | "N out of M months" must match the actual saving consistency record |
| Tax filing status | Language about filing being current/lapsed must match the actual filing status |

All the above checks also run against the **overall summary** section (not just per-metric lines).

### What is checked — Insight cards

Each insight card is checked against the data for its specific theme:

- Every rupee amount or percentage must trace to the theme's data
- Credit scores must match the score history
- "N out of 5" (tax) and "N out of M" (savings) patterns must match actual values

### Rupee amount cross-check (Summary only)

As a final safety net, every `₹` amount anywhere in the summary output is checked against a pool of all known amounts from the input (income, spend, EMI, investments, emergency corpus, insurance covers). If an amount can't be explained as an exact value, a near match, or a small combination of known amounts, it's flagged.

## 3. Trend & Direction Accuracy

**Why it matters:** Saying a metric "improved" when it actually worsened is misleading. These checks catch contradictions between what the data shows and what the text says.

**Applies to:** Summary and Insight cards

| Metric | What is checked |
|--------|----------------|
| Spending | If spend share rose over the period, the text must not say it "eased". If it fell, must not say "worsening". |
| Credit score | If the score dropped, text must not say it's "climbing" or "improving". If it rose, must not say it's "declining". |
| EMI burden | If EMI share increased, text must not say it "eased". If it decreased, must not say it's "worsening". |
| Investment rate | If the rate fell, text must not say it "accelerated". If it rose, must not say there was "slippage". |

For insight cards, similar direction checks apply to themes like credit score trends, EMI pressure, and investment momentum.

---

## 4. Domain Logic Consistency (Insight cards only)

**Why it matters:** Beyond number accuracy, insights must make logical sense given the data. These checks catch contradictions in meaning.

| Check | What it prevents |
|-------|-----------------|
| Spend exceeds income | Must not claim spending exceeds income when the spend ratio is well below the threshold |
| Cover gap when adequate | Must not claim a life/health insurance gap when coverage is already adequate (ratio >= 1.0) |
| Tax filing status | Must not say "filing is up to date" when the data shows it is not, and vice versa |
| Portfolio concentration | When one asset class dominates (>50%), the insight should acknowledge the concentration |
| Tax regime mismatch | Old-regime tax concepts (Section 80C, 80D, NPS) must not appear when the user is on the new tax regime |
| Subscription data | Must not cite subscription amounts when all subscription values are zero |

## 5. Content Quality Standards

**Why it matters:** Even if numbers are correct, the text must be well-formed, concise, and useful.

### Summary

| Rule | Limit |
|------|-------|
| Each metric summary | Maximum 25 words |

### Insight cards

| Rule | What it means |
|------|---------------|
| Must include a number | Every insight description must contain at least one concrete data point (rupee amount, percentage, or number) |
| No vague-only text | A description that's entirely qualitative with zero data points is rejected |
| No throwaway lines | Generic placeholders like "Your spending is around ₹X" without any analysis are rejected |
| Not too dense | Descriptions exceeding 29 words, 320 characters, 3 monetary mentions, or 3 sentences are rejected |
| Headline-description alignment | The headline and description must share at least one topical keyword |
| CTA alignment | The call-to-action must relate to the headline or description topic |

---

## 6. Text Cleanliness

**Why it matters:** Internal system artefacts, technical formatting, or AI reasoning traces must never reach the user.

**Applies to:** Summary and Insight cards

| Check | What it catches |
|-------|----------------|
| Internal jargon | The word "deterministic" (internal system term) must not appear |
| Population comparisons | Phrases like "most people", "typical earners", "compared to other users" are not grounded in the user's data |
| Word-count annotations | Editor debris like "(20-25 words)" or "Word count: 23" must be stripped |
| Internal field names (Insights) | Technical names like "spend_to_income_ratio" or "emi_burden" must not leak into user-facing text |
| Markdown formatting (Insights) | Bold markers, headings, bullet lists, or code formatting must not appear |
| AI reasoning traces (Insights) | Phrases like "Based on the data", "Looking at the JSON", "Let me analyse" must not appear |

---

## 7. Deduplication (Insight cards only)

**Why it matters:** The user should not see two insight cards saying essentially the same thing.

| Check | What it does |
|-------|-------------|
| Headline overlap | If two cards within the same pillar share 80%+ of the same keywords in their headlines, the later one is dropped |
| Same-amount dedup | If two cards cite the exact same set of monetary amounts, the later one is dropped |

---

## 8. Persona Consistency (Summary only)

When a user persona is available (based on their financial profile), the summary text is checked for language that contradicts the persona context. For example, advice-like phrasing inappropriate for a specific income bracket or life stage would be flagged.
